Spec — EN-19042: Portfolio benchmark CSV export

Goal

Export the benchmark widget's current view to CSV, honoring the active filters, date range and comparison set.

Approach

  1. Add ExportButton to BenchmarkHeader (read-only safe)
  2. Reuse benchmarkSelector to read the current view state
  3. Stream rows via toCsv() — chunk above 5k rows

Acceptance criteria

  • Button visible for all roles incl. read-only
  • Filename benchmark_{portfolio}_{date}.csv
  • Unit + e2e coverage for filtered export

Open questions — need an engineering decision

  1. Where should the CSV be generated — client, BFF, or batch service?
  2. How do we keep a 200k-row export from exhausting memory?
  3. Where should the export logic live to stay reusable & testable?
